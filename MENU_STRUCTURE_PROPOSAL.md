# 메뉴 구조 개선 제안서

## 현재 구조의 문제점

### 1. 관련 기능 분리
- 팝업 관리와 팝업 이미지 관리가 별도 메뉴로 분리
- 게임 규칙관리와 게임 데이터 관리가 별도 메뉴로 분리
- 관련 작업 시 여러 메뉴를 왔다갔다 해야 함

### 2. 메뉴 깊이 문제
- 게임 데이터 관리 하위에 4개 게임이 각각 메뉴로 존재
- 메뉴가 너무 깊어져서 한눈에 파악하기 어려움

## 제안하는 구조 (3가지 옵션)

### 옵션 1: 기능별 그룹화 (추천) ⭐
```
📊 대시보드
📢 팝업 관리
   ├─ 팝업 목록
   └─ 이미지 관리
🎮 게임 관리
   ├─ 게임 규칙
   └─ 게임 데이터 (게임 선택 드롭다운/탭)
📈 GA 이벤트 조회
```

**장점:**
- 관련 기능이 한 곳에 모여있어 작업 흐름이 자연스러움
- 메뉴 깊이가 적절함 (2단계)
- 확장성이 좋음 (새로운 게임 추가 시 메뉴 변경 불필요)

**단점:**
- 게임 데이터 페이지에서 게임을 선택해야 함

---

### 옵션 2: 평면 구조 (간단함)
```
📊 대시보드
📢 팝업 관리
📷 팝업 이미지 관리
🎮 게임 규칙 관리
📝 게임 데이터 관리 (게임 선택 드롭다운/탭)
📈 GA 이벤트 조회
```

**장점:**
- 모든 메뉴가 1단계로 접근 가능
- 구조가 단순함

**단점:**
- 관련 기능이 분산되어 있음
- 메뉴 항목이 많아짐

---

### 옵션 3: 하이브리드 구조
```
📊 대시보드
📢 팝업 관리
   ├─ 팝업 목록
   └─ 이미지 관리
🎮 게임 관리
   ├─ 규칙 관리
   └─ 데이터 관리
      ├─ 회화 퀴즈
      ├─ 연상 퀴즈
      ├─ 짝 맞추기
      └─ 단어 완성
📈 GA 이벤트 조회
```

**장점:**
- 각 게임별로 직접 접근 가능
- 관련 기능이 그룹화되어 있음

**단점:**
- 메뉴가 3단계까지 깊어짐
- 게임이 많아지면 메뉴가 길어짐

---

## 최종 추천: 옵션 1 (기능별 그룹화)

### 구현 방식
1. **팝업 관리 그룹**
   - 팝업 목록: 현재 팝업 관리 페이지
   - 이미지 관리: 현재 이미지 관리 페이지를 하위 메뉴로

2. **게임 관리 그룹**
   - 게임 규칙: 현재 게임 규칙관리 페이지
   - 게임 데이터: 
     - 단일 페이지에서 게임 타입을 탭 또는 드롭다운으로 선택
     - 선택한 게임의 데이터를 표시/추가/내보내기
     - URL: `/games/data?gameType=conversation-quiz` (쿼리 파라미터 방식)

### 게임 데이터 페이지 UI 제안
```
[게임 데이터 관리]
[회화 퀴즈] [연상 퀴즈] [짝 맞추기] [단어 완성]  <- 탭으로 게임 선택

[데이터 추가] [내보내기] [새로고침]

[데이터 테이블]
```

이렇게 하면:
- 메뉴 구조가 깔끔해짐
- 게임 추가 시 메뉴 변경 불필요
- 관련 기능이 한 곳에 모여있어 작업 효율 향상
